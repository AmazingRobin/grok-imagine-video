# 用户模板

这个目录用于创建自定义模板，以覆盖 Spec Workflow 的默认模板。

## 如何使用自定义模板

1. 在当前目录中创建自定义模板文件，文件名必须与要覆盖的默认模板完全一致：

   - `requirements-template.md`：覆盖需求文档模板
   - `design-template.md`：覆盖设计文档模板
   - `tasks-template.md`：覆盖任务文档模板
   - `product-template.md`：覆盖产品说明模板
   - `tech-template.md`：覆盖技术说明模板
   - `structure-template.md`：覆盖结构说明模板

2. 模板加载优先级：

   - 系统会先检查当前 `user-templates/` 目录
   - 如果这里存在匹配的模板，就使用该模板
   - 否则使用 `templates/` 目录中的默认模板

## 自定义模板示例

创建自定义需求模板：

1. 在当前目录中新建 `requirements-template.md`
2. 添加你的自定义结构，例如：

```markdown
# 需求文档

## 执行摘要
[你的自定义章节]

## 业务需求
[你的自定义结构]

## 技术需求
[你的自定义字段]

## 自定义章节
[添加适合你工作流的章节]
```

## 模板变量

模板可以包含占位符，创建文档时会自动替换：

- `{{projectName}}`：项目名称
- `{{featureName}}`：正在编写规格的功能名称
- `{{date}}`：当前日期
- `{{author}}`：文档作者

## 最佳实践

1. 从默认模板开始：复制 `../templates/` 中的默认模板作为起点
2. 保持结构一致：保留相近的章节标题，以确保工具兼容性
3. 记录变更：添加注释说明为什么新增或修改章节
4. 纳入版本控制：将自定义模板提交到版本控制中
5. 充分测试：确保自定义模板能和 spec workflow 工具正常配合

## 说明

- 自定义模板是项目专属内容，不包含在包发布内容中
- `templates/` 目录包含默认模板，并会随版本更新
- 当前目录中的自定义模板会在更新过程中保留
- 如果自定义模板存在错误，系统会回退使用默认模板
